Type: `os_type`

The os_type data source automatically detects the host operating system, version,
and CPU architecture where Packer is running. This is useful for making
OS-specific build decisions, tagging images with build environment information,
or conditional logic based on the build host.

## Configuration

This data source requires no configuration. It automatically detects the host
system information when executed.

## Output

- `os` (string) - The host operating system name (e.g., `darwin`, `linux`, `windows`)
- `version` (string) - The OS version/release number
- `architecture` (string) - The CPU architecture (e.g., `amd64`, `arm64`, `386`)
- `platform` (string) - Detailed platform information (e.g., `darwin`, `ubuntu`, `rhel`)
- `family` (string) - OS family classification (e.g., `Standalone Workstation`, `debian`, `rhel`)

## Example Usage

### Basic Usage

```hcl
data "host-info" "current" {
  # No configuration required
}

locals {
  build_info = "Built on ${data.host-info.current.os_type} ${data.host-info.current.version}"
}
```

### Conditional Logic Based on Host OS

```hcl
data "host-info" "current" {}

locals {
  # Use different settings based on the host OS
  docker_host = data.host-info.current.os_type == "darwin" ? "unix:///var/run/docker.sock" : "unix:///var/run/docker.sock"

  # Create OS-specific image tags
  image_tag = "myapp-${data.host-info.current.os_type}-${data.host-info.current.architecture}"
}

source "docker" "example" {
  image  = "ubuntu:22.04"
  commit = true
  changes = [
    "LABEL built_on_os=${data.host-info.current.os_type}",
    "LABEL built_on_arch=${data.host-info.current.architecture}",
    "LABEL build_platform=${data.host-info.current.platform}",
  ]
}

build {
  sources = ["source.docker.example"]
}
```

### Use in Output Naming

```hcl
data "host-info" "current" {}

build {
  sources = ["source.amazon-ebs.example"]

  post-processor "manifest" {
    output = "manifest-${data.host-info.current.os_type}.json"
  }
}
```
